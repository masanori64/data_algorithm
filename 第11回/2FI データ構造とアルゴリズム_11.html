線形探索，二分探索

探索

探索（searching）とは，nn個のデータが登録されている表の中からある特定の値を持つデータを探し出す操作です． 例えば，学籍番号から氏名を探す，といった操作が典型例となります． 探索は，最も頻繁に利用されるアルゴリズムの1つです．

表に格納する1つ1つのデータはレコード構造（record structure）になっています． このレコード構造は，1個以上のフィールド（field）を持っています． これらのフィールドのうち，ある特定のフィールドが探索の対象となります． 探索の対象となるフィールドをキー（key）と呼びます． Javaの場合，レコード構造はオブジェクト，フィールドはそのままフィールドが対応します．

つまり，探索は次の3つの機能から構成されます．

  1. 探索「与えられたキーをもつデータを表から探し出す」
  2. 挿入「データを表に登録する」
  3. 削除「データを表から取り除く」

線形探索

線形探索（linear search）は，配列に登録されたデータを先頭から順に調べていく方法です． 探索は，配列の先頭から1つ1つ要素を比較していき，目的のデータを探し出します． また，表を実現するデータ構造として，連結リストを使うこともできます．

線形探索を行う擬似コードは以下のようになります． 

class Entry {
  int key;     // キー
  Object data; // データ
}

Entry[] table = new Entry[n];  // 表の要素はn個

Object linearsearch(int key) {
  int i = 0;                 // (1)
  while (i < n) {            // (2)
    if (table[i].key == key) // (3)
      return table[i].data;  // (4)
    i++;                     // (5)
  }
  return null;               // (6)
}

ここで，線形探索の計算量を考えてみましょう． linearsearchメソッドの(1)は要素数nnによらず，1回実行されます． (2)で(3)と(5)が繰返し実行されます． 繰返しの過程で，目的のキーが見つかった場合は(4)が，見つからなかった場合は(6)が，1回だけ実行されます． ここで，(2)のwhile文は要素数nnに対して，何回繰り返されるのでしょうか． 最良の場合は1回目で目的のキーが見つかりますが，最悪の場合はキーが見つからないのでnn回実行されます． そのため，(2)，(3)，(5)は平均でn/2n/2回実行されることになります． linearsearchメソッドの各文の計算量をまとめると，以下の表となります．

文 実行回数 計算量
(1) 1 O(1)O(1)
(2) n/2n/2 O(n)O(n)
(3) n/2n/2 O(n)O(n)
(4) 1 O(1)O(1)
(5) n/2n/2 O(n)O(n)
(6) 1 O(1)O(1)

では，これらをまとめたlinearsearchメソッドの計算量はどうなるでしょうか． linearsearchメソッドは(1)～(6)の文が実行されますので，それぞれの文の計算量の和となります． ここで，計算量O(f(n))O(f(n))，O(g(n))O(g(n))の2つの操作を連続して実行する場合の計算量は，以下の通りです．

O(f(n))+O(g(n))=O(max(f(n),g(n)))O(f(n))+O(g(n))=O(max(f(n),g(n)))

つまり，線形探索のlinearsearchメソッドの計算量はO(n)O(n)となります．

二分探索

線形探索は計算量がO(n)O(n)の探索アルゴリズムですが，二分探索（binary search）はより性能の良い探索アルゴリズムです． 二分探索は，あらかじめキーの昇順（または降順）に整列されている配列の中から，特定のキーを持つデータを探し出すアルゴリズムです． ここでは，昇順で整列しているとします．

探索を行うには，まず探し出したいキーxxを，配列の中央の要素のキーyyと比較します．x==yx==yなら探索は成功です． もし，x<yx<yなら目的の要素は配列の前半にあるはずなので， 探索する範囲を配列の前半にして，再び比較から繰り返します． また，x>yx>yなら目的の要素は配列の後半にあるはずなので， 探索する範囲を配列の後半にして，再び比較から繰り返します． 範囲が要素１個になってもキーが一致しなければ探索は失敗です．

二分探索の擬似コードは以下のようになります． 

class Entry {
  int key;     // キー
  Object data; // データ
}

Entry[] table = new Entry[n];      // 表の要素はn個

Object binarysearch(int key) {
  int low = 0;                     // (1)
  int high = n-1;                  // (2)
  
  while (low <= high) {            // (3)
    int mid = (low + high) / 2;    // (4)
    if (key == table[mid].key)     // (5)
      return table[mid].data;      // (6)
    else if (key < table[mid].key) // (7)
      high = mid - 1;              // (8)
    else
      low = mid + 1;               // (9)
  }
  return null;                     // (10)
}

binarysearchメソッドではlow, high, midの3つの変数が出てきます． lowは着目している下限を表し，highは上限を表します． そして，着目している範囲の中央，つまり(low+high)/2をmidで表しています． このmidのキーと探索対象のkeyを比較することで，探索を行います．

midのキーとkeyが一致するならば，目的のkeyが見つかったことになります． もし，keyよりもmidのキーが大きい場合，目的のkeyはmidよりも前にあることが期待されるので，着目する範囲を前半分になるように，highの値を変更します． 逆に，keyよりもmidのキーが小さい場合，目的のkeyはmidよりも後にあることが期待されるので，着目する範囲を後半分になるように，lowの値を変更します． これを繰り返すことで，目的のkeyを探索することができます．

では，二分探索の計算量を考えてみましょう． 線形探索と同様に，binarysearchメソッドの各文の計算量は以下の通りになります．

文 計算量
(1) O(1)O(1)
(2) O(1)O(1)
(3) O(logn)O(log⁡n)
(4) O(logn)O(log⁡n)
(5) O(logn)O(log⁡n)
(6) O(1)O(1)
(7) O(logn)O(log⁡n)
(8) O(logn)O(log⁡n)
(9) O(logn)O(log⁡n)
(10) O(1)O(1)

(3)～(9)の繰り返しに注目しましょう． この繰り返しの計算量は(4)～(9)の合計となりますが，それぞれの処理は基本操作のみであり，要素数nnに依存しません． よって，(4)～(9)の計算量はそれぞれO(1)O(1)となります． つまり，繰り返し1回あたりの計算量はO(1)O(1)です．

では，この繰り返しは何回実行されるのでしょうか． 二分探索は繰り返しが1回実行されるごとに，着目する範囲が半分になります． つまり，繰り返しの回数は要素数nnに対して，log2n(=lgn)log2⁡n(=lg⁡n)回程度になります． 繰り返し1回あたりの計算量がO(1)O(1)の処理をlgnlg⁡n回行うので，繰り返し全体の計算量はO(logn)O(log⁡n)となります．

結果的に，二分探索の計算量はO(logn)O(log⁡n)となります． 線形探索の計算量がO(n)O(n)でしたが，この差はどのくらいあるのでしょうか． 影響が大きい繰り返しの部分について着目して考えてみましょう． 線形探索の繰り返しは最悪でnn回（平均でn/2n/2回）実行されるのに対して，二分探索では最悪でlgnlg⁡n回実行されます． ここでn=10000n=10000と仮定すれば，線形探索は10000回（平均で5000回）なのに対して，二分探索はわずか14回程度になります． これはnnが大きくなればなるほど顕著な差になっていきます．

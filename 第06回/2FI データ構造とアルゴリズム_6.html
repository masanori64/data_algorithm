リスト，再帰

リスト

リストとは同じ型のデータが並んだデータ構造です． セットと違って，並び順に意味があります． リストの例としては，ArrayListやLinkedListがありますが，ここではキューやスタックにも使われるLinkedListについて取り扱います．

連結リスト

連結リストとはリストに含まれる各要素をリンクによってつなぎ合わせたものです． 連結リストでは，各要素に値だけではなく次の要素へのリンクも持たせます． また，先頭の要素を表すための要素も準備します． 

class Node {
   Node next; // 次のノードへのリンク
   int data;  // 値（型は場合によって異なる）
}

Node head;    // 先頭ノードへのリンク

ノードが終端の場合，nextはnullとします． headは先頭のノードへのリンクを持ちますが，リストが空である場合はnullとします．

連結リストに対する操作を以下にまとめました．括弧内は対応するLinkedListのメソッドです．

  1. kk番目のノードの前にノードを挿入する（add）
  2. kk番目のノードを削除する（remove）
  3. kk番目のノードの内容を読む/書く（get, set）
  4. 特定のキーを持つノードを探索する（contains）
  5. 複数のリストを1つにまとめる（addAll）
  6. 1つのリストを複数のリストに分割する
  7. リストの複製を作る
  8. リストに含まれるノードの個数を求める（size）

連結リストへの挿入を行ってみましょう． 変数xで指定されているノードの次に，変数pで指されている新しいノードを繋ぎます． 変数pの次のノードを変数xの次のノードに変更し， 変数xの次のノードを変数pに変更することで，変数pのノードを変数xの後に挿入することができます． 

p.next = x.next;
x.next = p;

次に，連結リストからの削除を行ってみましょう． 変数xで指定されている次のノードを削除しましょう． 削除されるノードを変数pで指すようにすると，変数xの次のノードを変数pの次のノードに変更すれば，変数pのノードをリストから削除することができます． 

p = x.next;
x.next = p.next;

次に，連結リストのノードを順にアクセスすることを考えてみましょう． 変数pを使って順にアクセスをするためには，まず始めに，変数pは先頭のノードを指す必要があります． もし，変数pが指すノードが終端でないならば，変数pのデータを参照し，変数pを次のノードにして，同様の処理を繰り返します． つまり，擬似コードで表すと次のようになります． 

p = head;
while (p != null) {
  print(p.data);
  p = p.next;
}

ここで，配列と連結リストの計算量を考えてみましょう． 配列は任意の要素を参照することができます． これをランダムアクセス（random access）といいます． ランダムアクセスができるので，要素の参照に必要な計算量はO(1)O(1)です． 対して，連結リストはリンクを順番にたどるシーケンシャルアクセス（sequential access）しかできません． そのため，要素がnn個ある場合，要素の参照に必要な計算量はO(n)O(n)になります．

一方で，要素の挿入と削除については，連結リストの方が配列よりも高速に行えます． 配列でリストを表現していて，途中に要素を挿入する場合，それ以降の要素を全て後ろにずらさなくてはなりません． 同様に，削除する場合も，それ以降の要素を全て前にずらさなくてはなりません． この計算量はそれぞれO(n)O(n)になります． 対して，連結リストでは挿入と削除は一部のリンクを書き換えるだけですみます． これは要素数nnによらず，一定時間で行えますので，計算量はO(1)O(1)になります．

双方向リスト

連結リストは先頭から末尾に向かって順に辿ることしかできませんでしたが，前後に自由に辿ることができるようにしたものが双方向リストです． 双方向リストでは，各ノードは2つのリンクを持ち，前後のノードを指し示すことができます． 

class Node {
   Node prev; // 前のノードへのリンク
   Node next; // 次のノードへのリンク
   int data;  // 値（型は場合によって異なる）
}

Node head;    // 先頭ノードへのリンク
Node tail;    // 末尾ノードへのリンク

双方向リストの表現方法としては2つあります．

  1. 先頭と末尾を指すリンクによる表現法
  2. 循環リストによる表現法

1の方法では，先頭ノードを指すheadと末尾ノードを指すtailの2つの変数を用いて，双方向リストの先頭と末尾を表します． 双方向リスト中の先頭ノードはprev = nullとし，末尾ノードはnext = nullとします（Fig.5.10(a)）．

2の方法では，先頭ノードを指すheadのみを用います． 双方向リスト中の先頭ノードはprevで末尾ノードを指し，末尾ノードはnextとして先頭ノードを指すことで，循環リストとします（Fig.5.10(b)）． 空リストの場合は，nextもprevも先頭ノードを指します（Fig.5.10(c)）． 一般的な双方向リストの場合は，2の方法で実現されていることが多いです．

連結リストでは，ノードの挿入と削除は指定したノードの次のノードに限られていました． 対して，双方向リストではノードは前後のノードへのリンクを持っているので，挿入は前後いずれにも，削除については指し示したノード自体を削除できます． これは，双方向リストの大きな利点となります．

再帰

再帰（recursion）とは，「ある処理」のアルゴリズムを「ある処理」自身を使って実現する方法です． 再帰の具体例は以下の通りです．

階乗
f(n)={f(n−1)×n(n>0)1(n=0)f(n)={f(n−1)×n(n>0)1(n=0)
フィボナッチ数
f(n)={f(n−2)+f(n−1)(n>2)1(n=2||n=1)f(n)={f(n−2)+f(n−1)(n>2)1(n=2||n=1)
ユークリッド互除法
gcd(m,n)={gcd(n,mmodn)(n>0)m(n=0)gcd(m,n)={gcd(n,mmodn)(n>0)m(n=0)

階乗計算の再帰処理を考えてみよう． 5!5!を計算する場合，次のように展開できます．

  1. 5!=5×4!5!=5×4!
  2. 4!=4×3!4!=4×3!
  3. 3!=3×2!3!=3×2!
  4. 2!=2×1!2!=2×1!
  5. 1!=1×0!1!=1×0!
  6. 0!=10!=1

5!5!の計算をするとき，4!4!の値は確定していないので，計算結果を一旦保留します． ここで，一旦保留する際に，計算途中のデータや状態をスタックに積みます． 5!5!の計算結果を確定するために，4!4!を計算します． 4!4!の計算をするとき，3!3!の値は確定していないので，この計算結果も一旦保留します． 同様に繰り返していくと，0!=10!=1で初めて計算結果が確定します． 0!0!の結果が確定すると1つ前の計算が可能になるので，スタックからデータを取り出して計算すると，1!=1×0!=1×1=11!=1×0!=1×1=1の計算結果が得られます． 同様にして，2!=2×1!=22!=2×1!=2，3!=3×2!=63!=3×2!=6，4!=4×3!=244!=4×3!=24，5!=5×4!=1205!=5×4!=120と計算結果が順次確定します．

再帰処理は大きく分けて2つの部分からなります．

  1. 基本部分
  2. 再帰部分

再帰部分は自分自身を呼び出して，処理を継続する部分です． 再帰部分だけでは，いつまでも自分自身を呼び出し続けてしまい，永久に終わりません． そのため，再帰には終了条件が必ず必要になります． 終了条件によって，再帰処理を行わずに呼び出し元に戻り値を返す部分が基本部分です． 縮退と呼ばれることもあります．

再帰をプログラムで表現する場合として，ユークリッド互除法を考えてみましょう． 次のような再帰処理によるユークリッド互除法のメソッドを考えます． 

int gcd(int m, int n) {
  if(n=0) {
    return m; //基本部分
  } else {
    gcd(n, m%n); //再帰部分
  }
}

入力として，m=70,n=21m=70,n=21を考えてみましょう． ユークリッド互除法は以下のように動作します． 

gcd(70, 21)が呼び出される
  gcd(21, 70%21)が呼び出される（gcd(21, 7)）
    gcd(7, 21%7)が呼び出される（gcd(7, 0)）
      n=0なので，m=7が戻り値
    gcd(7, 21%7)の結果は7
  gcd(21, 70%21)の結果は7
gcd(70, 21)の結果は7

以上のようになります．

連結リストの参照を再帰で考えることもできます． 再帰の特徴は自分で自分を呼び出す関係にあります． 連結リストのノードも自分に自分を含んでいるため，再帰とよく似た表現になっています． 先に示した擬似コードをメソッド化すると以下のようになります． 

void follow() {
  p = head;
  while (p != null) {
    print(p.data);
    p = p.next;
  }
}

このfollowメソッドを再帰で表現すると以下のようになります． 

void follow_r(Node p) {
  if (p = null) {
    return;
  }
  print(p.data);
  follow_r(p.next);
}

このfollow_rメソッドはpがnullになるまで，pの値を表示してからpの次のノードを引数にして自分自身を呼び出します（再帰）． 末尾ノードの次のノードはnullになっているので，pがnullになれば再帰は終了します．

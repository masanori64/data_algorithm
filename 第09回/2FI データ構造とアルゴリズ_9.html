木

木（tree）は，階層関係を表現するためのデータ構造です． 階層関係とは，ある概念Aの上には概念Bがあり，概念Bの上にはさらに概念Cがあるといった，上下関係（親子関係）のことをいいます． 前回説明したように，木はグラフの一種です． 下図左は木ですが，右は木ではなくグラフです．

木の身近な例としては，組織図があります． 調べてみたところ，残念なことに，本学の組織図は木ではありませんでした・・・．

木は実世界だけではなく，コンピュータの世界でも広く利用されています． 例えば，ほとんどのOSで採用されている階層ディレクトリ構造は木構造そのものです． コンパイラが数式を処理する際は，数式を表す木を組み立ててから，その木を解析してオブジェクトコードを生成します． 逆ポーランド記法も木で表すことができます． 木はデータの表現だけではなく，計算過程としてみることもできます． バックトラック法はまさに木で表現されています．

木で用いる用語の説明をしましょう． 下図の丸で示されるものをノード（node）と呼びます． 節，節点，頂点（vertex）と呼ばれることもあります． 各ノードには名前（図中では数字）を付けて識別します． ノードには値を持たせることもできます． この値のことをラベル（label）と呼びます． ここで，ノードの名前とラベルを混同しないようにして下さい． リストに喩えるならば，名前は（何番目かという）セルの位置、ラベルはセルに格納されている値にそれぞれ該当します． ノード間の連結関係をエッジ（edge）と呼びます． 枝や辺と呼ばれることもあります． ノードとエッジはグラフと共通の用語ですが，木ではその特徴になぞらえて，節と枝と呼ぶことが多いです．

ノードの中から1つを選び，それを根（root）と呼びます． Fig.6.2の例では，ノード1が根になります． 階層関係はノードを配置する位置で表現します． Fig.6.2の例では，ノード1が最上位で，ノード2と3がその次の階層となります．

木ではノードの階層関係に意味があります． この階層関係を親（parent）と子（child）という用語で表現します． Fig.6.2の例では「ノード1はノード2と3の親」「ノード2はノード1の子」などといいます． ノードは1つの親しか持つことができませんが，根は例外で親を持ちません． ノードは任意の個数（0個でも構いません）の子を持つことができます． 子を持たないノードのことを葉（leaf）と呼びます． 外部ノード（external node）と呼ばれることもあります． Fig.6.2ではノード4, 6, 8, 9, 10が葉になります． 対して，葉ではないノードのことを内部ノード（internal node）といいます． Fig.6.2ではノード1, 2, 3, 5, 7が内部ノードになります． 内部ノードは子を持つノードですので，根も含まれる点に気をつけて下さい．

あるノードの子を根とする木を（元の木の）部分木（subtree）といいます． Fig.6.2では，ノード2を根とする部分木とノード3を根とする部分木の2つの部分木があります． さらに，ノード2を根とする部分木には，ノード4, 5, 6を根とする部分木があります．

同じ親を持つノード同士を兄弟（sibling）といいます． Fig.6.2では，3組の兄弟があります．

  1. ノード2とノード3
  2. ノード4とノード5とノード6
  3. ノード8とノード9

最後に，レベル（level，水準）を説明します． まず，木の根をレベル0と定義します． 次に，階層関係を子に向かって1回辿るごとに，レベルを1ずつ増やします． レベルとは，根からそのノードまでの距離と考えることができます． 根と根から1番遠い葉とのレベルの差を高さ（height）といいます． また，各ノードと根とのレベルの差を深さ（depth）といいます． Fig.6.2では，各レベルのノードは以下のようになります．

レベル0
ノード1
レベル1
ノード2，ノード3
レベル2
ノード4，ノード5，ノード6，ノード7
レベル3
ノード8，ノード9，ノード10

最後に，木の正式な定義を与えましょう． 木は次の2つの規則によって再帰的に定義される構造です．

  1. 1つのノードは，それ自体が木である．この木に含まれるただ1つのノードが，この木の根である．
  2. kk個の木T1…TkT1…Tkがあり，それぞれの根をノードn1…nkn1…nkとする．ノードnnをノードn1…nkn1…nkの親にすると，ノードnnを根とする新しい木TTが得られる．このとき，木T1…TkT1…Tkは，木TTの部分木であるという．部分木の根n1…nkn1…nkは，ノードnnの子であるという．

順序木

同じ親を持つノード同士を兄弟と定義しましたが，兄弟の順序付けを行った木が順序木（ordered tree）です． 対して，兄弟の順序付けを行わない木を無順序木（unordered tree）と呼びます． 一般に使われるデータ構造では，順序木を用いるのが典型的です． 順序木で兄弟を順序付けする場合，左に位置するノードを長子として，右にあればあるほど年下と見なすのが一般的です．

Fig.6.4(a)と(b)では，兄弟ノードであるノードbとノードcの位置が違います． 順序木として見た場合，Fig.6.4(a)はノードbが兄でノードcが弟になり，Fig.6.4(b)はノードcが兄でノードbが弟となるため，別の木となります． 無順序木として見た場合，兄弟の順序付けを行いませんので，Fig.6.4(a)と(b)は同じ木となります．

二分木

木構造の一種に，二分木（binary tree）があります． 二分木は次のように定義されます．

  1. 空の木は二分木である．
  2. 次のいずれかの条件を満たすノードのみからなる木は，二分木である．
    • 子を持たない
    • 左の子のみを持つ
    • 右の子のみを持つ
    • 左右2個の子を持つ

二分木ではノードは高々2個の子しか持ちません． また，子が1個の場合，左の子か右の子かを区別します． 左の子を根とする部分木を左部分木，右の子を根とする部分木を右部分木と呼びます． 二分木の中でも，全てのノードが葉であるか，または2つの子を持っているものを全二分木（full binary tree）といいます． また，全ての葉が同じ「深さ」を持ち，全ての内部ノードが2つの子を持っているような二分木を完全二分木（complete binary tree；狭義の完全二分木）といいます． さらに，優先的にレベルの小さいノードかつ左側のノードに子を配置した二分木をほとんど完全二分木（almost complete binary tree；広義の完全二分木）といいます． Fig.6.4は全二分木であり，完全二分木です． nn個のノードを持つ完全二分木の高さは一般に，⌊lgn⌋⌊lg⁡n⌋で表されます．

Fig.6.5の例で二分木を説明しましょう． Fig.6.5(a)はノードcがノードbの左の子ですが，Fig.6.5(b)はノードcがノードbの右の子になっています． そのため，Fig.6.5(a)と(b)は別の二分木となります． 二分木を描く際は，左右の子を区別する必要があるので，左の子は親の左斜め下に，右の子は親の右斜め下に描き，エッジで結びます．

二分木は高々2個しか子を持ちませんので，以下のようなクラスを考えることができます． 

class BinaryTreeNode {
  BinaryTreeNode left;  // 左の子ノード
  BinaryTreeNode right; // 右の子ノード
  String label;         // ラベル（型は場合によって異なる）
}

leftやrightがnullの場合は，子が存在しないことを意味します．

BinaryTreeNodeクラスでFig.6.6(a)の二分木を表現すると，Fig.6.6(b)のようになります．

データ列，スタック，キュー，セット，マップ

データ列

配列

配列（array）は，同じ型の要素を集めたものです． 配列の要素は，添え字（index）によって識別します． 

public class ArrayTest {
  public static void main(String[] args) {
    //配列の確保
    String[] array = new String[3];
    
    //要素の代入
    array[O] = "alice";
    array[1] = "bob";
    array[2] = "carol";
    
    //要素の参照
    for (int i = 0; i < array.length; i++) {
      System.out.println(array[i]);
    }
  }
}

配列における要素の参照，挿入，削除に必要な計算量を考えてみましょう． 配列では添え字を使うことで任意の要素を即座に参照することができます． そのため，要素の参照に必要な計算量はO(1)O(1)です． 対して，任意の位置に要素を挿入するには，その位置以降にある要素を全て後にずらす必要があります． そのため，それに必要な計算量はO(n)O(n)となります． 任意の位置にある要素の削除についても同様に，その位置以降にある要素を全て前にずらす必要があるため，その計算量はO(n)O(n)となります．

ArrayList

JavaではArrayListを使うことで要素を追加できる配列のようなものを作ることができます． 

import java.util.*;

public class ArrayListTest1 {
  public static void main(String[] args) {
    //ArrayListのインスタンスの確保
    ArrayList<String> list = new ArrayList<String>();

    //要素の追加（addメソッド）
    list.add("alice");
    list.add("bob");
    list.add("carol");
    
    //要素の参照（getメソッド）
    for (int i = O; i < list.size(); i++) {
      System.out.println(list.get(i));
    }
  }
}

ArrayListは追加，代入，参照以外にも，removeメソッドを使って要素を削除したり，containsメソッドを使って特定の要素を含んでいるか調べたりできます． ArrayListは要素の追加を高速に行うことができますが，末尾以外に要素を挿入したり，要素を削除するには時間がかかります． そのため，要素の挿入や削除を頻繁に行う場合は，ArrayListではなくLinkedListを使う方が良いです．

LinkedList

LinkedListは連結リストを内部構造に持つリストです． 連結リストとは，リストの要素1つ1つがオブジェクトとなっていて，オブジェクト間のつながりは参照関係（has-a関係）によって実現されています． LinkedListの場合には双方向リストになっていて，着目している要素の1つ後ろの要素や1つ前の要素が容易に取得できます．

LinkedListではArrayListが苦手な要素の挿入や削除を高速に行うことができます． 一方で，特定の位置にある要素を参照する処理は低速です． 

import java.util.*;

public class LinkedListTest1 {
  public static void main(String[] args) {
    //LinkedListのインスタンスを確保する
    LinkedList<String> list = new LinkedList<String>();

    //要素の追加（addメソッド）
    list.add("bob");
    list.add("carol");
    list.add("dave");
    list.add("eve");

    //現在の状態
    System.out.println(list);

    //先頭にAliceを挿入
    list.addFirst("alice");

    //Aliceを挿入した後の状態
    System.out.println(list);
  }
}

スタック

スタック（Stack）は「後入れ先出し（Last-In First-Out, LIFO）」を表すデータ構造です． 「先入れ後出し（First-In Last-Out, FILO）」と呼ばれることもあります．

現実世界でスタックとなる例を以下に示します．

  • 細長い袋状のもの
    • コップ
    • リュック
  • 積ん読（本を重ねて積んでいくと上からしか取れない）
  • 縦長の駐車場に車2台を入れる（先に停めた車は出られない）
  • 狭い冷蔵庫（前のものを取らないと昔入れた後ろのものが取れない）
  • 後から後からより重要度の高い仕事がやってくる状況（いまやっている仕事を棚上げ、の繰り返し）
  • UNIX系OSのpushd/popdコマンド

スタックを擬似コードで説明します． スタックにデータを追加する命令をpush，データを取り出す命令をpopとします． 

s.push("alice");
s.push("bob");
s.push("carol");
s.push("dave");
s.push("eve");

上記の擬似コードを実行後は，スタックsにはalice, bob, carol, dave, eveの順にデータが格納されています． この時，最初に追加されたaliceをbottom，最後に追加されたeveをtopと呼びます． この状態で，以下の擬似コードを実行すると，eveが取り出され，スタックsにはalice, bob, carol, daveが残ります． 

s.pop();

スタックにデータを追加する場合は，常にtopに追加されるので，pushされた順番にデータが並びます． データを取り出す場合は，最後にpushされたデータ，つまりtopのデータが取り出されます．

スタックを配列で実現することを考えます． 配列xを使うとすると，x[0]がスタックのbottom，x[n-1]がtopになります． ここで，nはスタックに入っている要素の個数です． 変数nは，スタックポインタ（stack pointer）と呼ばれます． スタックポインタの値は，要素を積むとき（push）には1つ増加し，要素を降ろすとき（pop）には1つ減少します． スタックにデータを積むには，x[n++] = data;とします． スタックからデータを降ろすには，data = x[--n];とします． このように，スタックでは要素の挿入と削除が計算量O(1)O(1)で済みます．

スタックとして使う配列の大きさは有限ですから，要素を積む余裕があるかどうかをチェックする必要があります． また，スタックからデータを降ろす際には，スタックが空でないことを確認しなければなりません． これらのチェックは，スタックポインタの値を調べるだけなのでO(1)O(1)で済みます． 結局，チェックの有無にかかわらず，挿入と削除の計算量はO(1)O(1)になります．

Javaではスタックを表すクラスStackがあります． 追加はpush(E item)，取り出しはpop()というメソッドを用います． なお，StackクラスはArrayListに似たVectorクラスを継承していますので，配列と同じような性質を持ちます．

新しいJavaではLinkedListがスタックを実現するクラスになります． Java1.6以降では，両端から要素の追加/取り出しが可能なDeque（double ended queue，デック）インターフェイスがありますので，LinkedListクラスをDequeインタフェースで使うのが良いでしょう． Dequeインターフェースでスタックを実現するときは，push, popメソッドを用いるのが良いでしょう．

スタックの使用例として逆ポーランド記法（reverse polish notation）を説明します． 逆ポーランド記法は演算子を後ろに置き，値の保持にスタックを使います． 例えば，(1+2)∗3(1+2)∗3は1 2 + 3 *となります． 逆ポーランド記法の計算は以下のように行われます．

  1. 数値であれば，そのままスタックにpushする．
  2. 演算子であれば，スタックから2つの値をpopして演算を行い，演算結果をスタックにpushする．

最終的な演算結果はスタックのtopに保持される．

キュー

キュー（Queue）は「先入れ先出し（First-In First-Out, FIFO）」を表すデータ構造です． キューからデータを取り出すときは，先に格納したものから順に取り出されます．

現実世界でキューとなる例を以下に示します．

  • 筒状のもの
    • パイプ
    • ホース
    • ところてんを作る器具
    • CPUのパイプライン
    • 一方通行で追い越しできない道
  • 銀行や病院の待ち行列（先に並んだ人からサービスを受ける）
  • コンビニ店員の理想とする商品管理（先に入荷したものから売る; 後ろから補充）

キューを擬似コードで説明します． キューにデータを追加する命令をenqueue，データを取り出す命令をdequeueとします． 

q.enqueue("alice");
q.enqueue("bob");
q.enqueue("carol");
q.enqueue("dave");
q.enqueue("eve");

上記の擬似コードを実行後は，キューqにはalice, bob, carol, dave, eveの順にデータが格納されています． この状態で，以下の擬似コードを実行すると，aliceが取り出され，キューqにはbob, carol, dave, eveが残ります． 

q.dequeue();

キューにデータを追加する場合は，常に末尾に追加されるので，enqueueされた順番にデータが並びます． また，データを取り出す場合も，enqueueされた順番に取り出されます．

キューを配列で実現することを考えます． スタックの場合には，挿入と削除が同じ端で行われるので，要素の増減に合わせてスタックポインタを動かすだけで済みます． しかしキューの場合は，挿入と削除が反対の端で行われるので，キューの先頭を指すfrontと末尾を指すrearを用意します． キューに何も要素が入っていない状態では，front = rearとなります． キューに要素を入れるごとにrearが1つ増え，要素を取り出すごとにfrontが1つ増えます． キューの挿入，削除はともに計算量O(1)O(1)で実行することができます． ただし，frontもrearも増加する一方なので，無限大の配列が必要になります．

そこで，配列をリング状にして，「最後の要素の次には最初の要素がある」と考えることにします． つまり，n個の要素を持つ配列xにおいて，最後の要素x[n-1]の次に最初の要素x[0]が続くと考えます． 配列をリング状に扱うには，frontやrearを1つ増加させるときに，剰余をとります． このようにリング状になった配列をリングバッファ（ring buffer）といいます．

ところで，キューが満杯になったとき，最後の要素の次の要素が，キューの最初の要素になります． つまり，キューが満杯のときには，rear = frontとなります． しかし，rear = frontは待ち行列が空であることを示すので区別がつきません． これを解決するには，以下の3通りの方法があります．

  1. キューが空であることを示すフラグを別途用意する
  2. 必ず空の要素を1つ残しておく
  3. キューに入っている要素の個数を記録しておく

JavaのCollections Frameworkでは，キューを表すQueueインタフェースがLinkedListに実装されているので，LinkedListをキューとして使うことができます． 擬似コードで示したenqueueとdequeueに対応するメソッドは，それぞれofferとpollになります．

セット

セットは「要素の集合」を管理するインターフェースです． 集合の特徴は以下の通りです．

  • 要素には順序がない．
  • 要素には重複がない．

JavaのCollections Frameworkでは，集合を表すSetインタフェースが用意されています． HashSetクラスは，Setインタフェースを実装している代表的なクラスです． ListやMapインターフェースは重複を許しますが，Setインターフェースでは重複を許しません．

セットはリストと比較して，containsメソッドやremoveメソッドが高速に実行できます． リストの場合には先頭から線形探索をしていく必要があるため低速な処理になりますが，HashSetではハッシュ法（第12回で学びます）によって直接アクセスすることができるため，高速に動作します． 要素の順序づけが必要なく，あるオブジェクトが集合に含まれているかを確認することが多い場合には，リストではなくセットを用いるとよいでしょう．

マップ

マップは「キーと値の対応関係」を管理するインタフェースです． 最近の高級言語では，連想配列や辞書（dictionary），マップなどとして，標準で提供されることが多くなっています． データベースの世界では，key-valueストア（Key-Value Store, KVS）と呼ばれることもあります．

マップでは，キーを指定して，それに対応する値を得ます． 例えば，キーとして学籍番号，値として氏名を登録すれば，学籍番号から氏名を得ることができます．

キー 値
12FI888 電大太郎
12FI889 電大次郎
12FI890 電大三郎
12FI892 電大五郎
12FI893 電大花子
12FI895 電大次子

キーは重複を許しませんが，異なるキーであれば同じ値を格納することができます． そのため，以下のようなマップを作ることもできます．

キー 値
ロイヤルペンギン ペンギン目ペンギン科マカロニペンギン属
イワトビペンギン ペンギン目ペンギン科マカロニペンギン属
ジェンツーペンギン ペンギン目ペンギン科アデリーペンギン属
フンボルトペンギン ペンギン目ペンギン科フンボルトペンギン属
コウテイペンギン ペンギン目ペンギン科コウテイペンギン属

JavaではMapインタフェースと，その実装であるHashMapクラスがあります． HashMapでは，キーから値を高速に取り出すアルゴリズムとして，ハッシュ法を用いています． Javaではキーも値もオブジェクトであればなんでもかまいません．

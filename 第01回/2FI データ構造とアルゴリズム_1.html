アルゴリズムと計算量

アルゴリズムとは

アルゴリズムとは，ある問題を解くための手順のことです． コンピュータにアルゴリズムを実行させるためには，プログラミング言語でアルゴリズムを記述しなくてはなりません． プログラミング言語では，条件分岐，繰り返しなどを組み合わせながら，アルゴリズムを記述します． アルゴリズムをプログラミング言語で表現したものがプログラムになります．

アルゴリズムは元々，数学上の問題の解法を意味していました． 記録に残る最古のアルゴリズムは，ユークリッドの原論に書かれているものです． その中でも，二つの整数の最大公約数を求めるユークリッド互除法は，典型的なアルゴリズムとして知られています． 「二つの整数の最大公約数を求めよ」という問題は，以下のように定義できます．

  • 入力：2つの正の整数n,mn,m
  • 出力：nnとmmの最大公約数

この入力からどのようにして出力を生成するかについての手順， すなわち，この問題を解決するためのアルゴリズムの一つに，ユークリッド互除法があります． ユークリッド互除法は以下の通りです．

  1. 入力をm,n(m≥n)m,n(m≥n)とする．
  2. n=0n=0なら，mmを出力してアルゴリズムを終了する．
  3. mmをnnで割った余りを新たにnnとし，さらに元のnnを新たにmmとして，2.に戻る．

このように，ある問題に対し，その解き方を手順化して与えたものがアルゴリズムです． アルゴリズムは上記のように自然言語を用いて表すこともできますが，より形式的な表現形式として， 疑似コードを用いて記述されることがあります．ユークリッド互除法の疑似コード（Ｃ言語風）の例は以下の通りです． 

euclid(n, m){
  if(n > m) nとmを交換;  //大きい方の数をmとする
  while(n != 0){
    r = m%n;
    m = n;
    n = r;
  }
  return m;
}

練習01-1

上記の「二つの整数の最大公約数を求めよ」という問題を解く最も素朴なアルゴリズムの一つとして， 次のようなアルゴリズムを考えることができます．

  1. 1, 2, 3, ... の順にnnを割り切れるか，mmを割り切れるかをチェック
  2. これをnnとmmのうちの小さいほうの数に到達するまで行い， nnとmmの両方を割り切ることができた数のうち，最大のものを出力
このアルゴリズムの疑似コードを考えてみましょう．
ヒント： 変数nnが変数iiで割り切れることは，n%i==0n%i==0で表すことができる．

プログラムにおける重要な要素として「アルゴリズム」の他に「データ構造」があります． N. Wirth（ヴィルト）は，アルゴリズムとデータ構造とプログラムの関係を以下のように表現しています．

アルゴリズム＋データ構造＝プログラム

コンピュータで計算を行う場合，データを何らかの形式でメモリ等に記憶しておく必要があります． その形式のことをデータ構造と呼びます． 最も単純なデータ構造は配列でしょう． 代表的なデータ構造としては，リスト，木，グラフなどがあります．

数学的には，アルゴリズムの実行速度にあまり関心は払われません． しかし，コンピュータが実行するには，時間が必ずかかります． また，扱うデータの個数によっても，実行時間は大きく左右されます． したがって，有限の処理能力しかないコンピュータを使って問題を解くには，アルゴリズムの性能，つまり計算量を意識する必要があります．

計算量

アルゴリズムの性能は，計算量（complexity）という尺度で表されます． 計算量とは，仮想的なコンピュータによって，あるアルゴリズムを実行したときに，どれくらいの時間がかかるかを，入力の大きさnnの関数として表現したものです． 計算量は，コンピュータのハードウェア，プログラミング言語，コンパイラの性能，プログラマの技量などに影響を受けない表現でなければなりません． アルゴリズムの計算量は以下の二つで表されます．

  • 時間計算量（time complexity）
  • 領域計算量（space complexity）

同じ結果が得られるのであれば，時間計算量，領域計算量ともに小さいアルゴリズムが良いアルゴリズムとなります． なお，今後は単に「計算量」とした場合，時間計算量を指すものとします．

実行時間が入力の大きさnnの二乗に比例するアルゴリズムを「実行時間がO(n2)O(n2)のアルゴリズム」と呼びます． O(n2)O(n2)は「n2n2のオーダー」または「オーダーn2n2」と読みます． このような計算量の表現をOO記法と言います．

OO記法ではいくつかのルールがあります．

  • 発散速度が最も速い項でオーダーを表す
  • 係数は1とする

例えば，ある関数f(n)f(n)がf(n)=4n3+2n2+100n+1f(n)=4n3+2n2+100n+1である時，この計算量はO(n3)O(n3)となります． そのため，OO記法は「大雑把」と言われることもあります．また，OO記法は，そのアルゴリズムを用いて問題を解く場合に，その計算量がそれより大きくなることはないこと（計算量の上界）を意味しています．（この講義では扱いませんが，OO記法とは逆にアルゴリズムの計算量がそれより小さくなることはないこと（計算量の下界）を表すものに，ΩΩ記法がある．）

計算量を求めるにあたっては，最悪の入力データを想定します． これを最大計算量（worst-case complexity）と呼びます． これに対して，全ての入力データの対する計算量の平均をとったものを平均計算量（expected complexity）と呼びます． 普通は，計算量といえば，最大計算量を意味します．

計算困難な指数時間アルゴリズムを多項式時間アルゴリズムにすることができれば，大幅に計算量を削減できます．

先ほど例示したユークリッド互除法の計算量について考えてみます． ラメの定理によれば，割って余りを取るという操作を，最悪でも小さい方の十進法での桁数の約五倍繰り返せば，最大公約数に達します． ここで，入力された二つの整数のうち小さい方の整数nnの十進法での桁数をddとすれば，O(d)O(d)回の除算で最大公約数が求められます． 桁数ddはlognlog⁡nで求められるので，ユークリッド互除法の計算量はO(logn)O(log⁡n)となります．

冒頭の疑似コードをもとにユークリッド互除法の計算量を考えると次のようになります． m=nq+rm=nq+rとq≥1q≥1より，m−r=nq≥n>rm−r=nq≥n>rとなり，r<m/2r<m/2が成り立つ． rrの値はループの2回の繰り返しの後に新たなmmとして扱われるため，ループを2周するごとに，mmの値を半分未満の値にできることになります．したがって，最も回数が多い場合でも，2⌈log2m⌉−12⌈log2⁡m⌉−1回の繰り返しを行えば，r=0r=0となり，計算が終了します．したがって，上述した通り，ユークリッド互除法の計算量はO(logn)O(log⁡n)となります．

アルゴリズムの計算量

例えば，次のような1～nnまでの整数の総和を求めるアルゴリズムを考えましょう． 

int sum(int n) {
  int s=1;
  for(int x=2; x≤n; x++) {
    s+=x;
  }
  return s;
}

このアルゴリズムの計算量がどうなるか考えてみましょう． まず，入力の大きさはnnになります． メソッドsumの内部では，s+=xがfor文でn−1n−1回繰り返されています． そのため，計算量はO(n)O(n)の線形時間になります．

ところで，同じ1～nnまでの整数の総和を求める方法は他にもあります． 次のようなアルゴリズムを考えましょう． 

int sum2(int n) {
  return n*(n+1)/2;
}

さて，このアルゴリズムの計算量はどうなるでしょうか． 入力の大きさはnnですが，nnの大きさによらず，メソッドsum2の内部では，n*(n+1)/2が1回実行されるだけです． 細かくは，かけ算が1回，足し算が1回，割り算が1回ですが，OO記法ではO(1)O(1)の定数時間計算量（このような計算量を定数オーダーという）になります．

このように，同じ目的を持つアルゴリズムであっても，アルゴリズムや実装方法によって，計算量が異なることがあります．

余談：対数について

数学で学んだ対数の表記は，例えばlog25log2⁡5等であったと思います． しかし，この講義に限らず，一般的には底を表記せず，lognlog⁡nのように書くことが多いです． これは何故なのでしょうか．

ここで，底の変換公式を思い出しましょう． 底の変換公式は以下の通りです．

底の変換公式
logab=logcblogcaloga⁡b=logc⁡blogc⁡a

つまり，底の変換公式によれば，底10の対数を底eeの対数に変換する場合，以下のようになります．

底10を底eeに変換する方法
log10b=logebloge10log10⁡b=loge⁡bloge⁡10

よって，底の変換を行うことは，高々，係数が掛かる程度に過ぎません． 計算量では，先に説明したように，係数は気にしませんので，対数の底がいくつであるのかというのは問題になりません．

なお，対数の表記方法にはいくつかあるので，知っておくと役に立つことがあると思います．

底がeeの場合
logex=lnxloge⁡x=ln⁡x
底が2の場合
log2x=lgxlog2⁡x=lg⁡x

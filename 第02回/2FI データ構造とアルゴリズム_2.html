コレクションによるソート

ソート

ソート（整列，sorting）とは，レコードの集まりをキーの値の大小関係に従って並べ換える操作です． ソートは最も頻繁に行われる処理の一つです． キーが小さいものから大きなものへと，単調増加になるように並べることを昇順（ascending order）ソート，キーが大きなものから小さなものへと，単調減少になるように並べることを降順（descending order）ソートといいます． データの中に同じキー値を持ったレコードが2つ以上存在しても構いません． その場合，整列した結果，同じキーを持ったレコードが連続して並びます．

キーの大小を比較して並び替えるソートアルゴリズムは，nn個のレコードをソートするのに，最小でもO(nlogn)O(nlog⁡n)の計算量が必要となることが証明されています． 計算量がO(nlogn)O(nlog⁡n)のソートアルゴリズムとして，クイックソート，マージソートなどがあります．

次回は単純なソートとして，計算量がO(n2)O(n2)のソートアルゴリズムとして，バブルソート，選択ソート，挿入ソートを紹介します． 次々回は計算量がO(nlogn)O(nlog⁡n)のソートアルゴリズムとして，クイックソートとマージソートを紹介します．

ソートアルゴリズムの動作を直感的に理解するには，可視化が重要な役割を果たします． 以下に，各種ソートアルゴリズムを可視化した例を紹介します．

ソートアルゴリズムの計算量

上記では，各種アルゴリズムの計算量について，nn個のレコードをソートするのに必要な計算量という尺度を使って紹介してきました． しかし，どのソートアルゴリズムを選択するのかを判断する際には，もう少し深く考える必要がある場合があります．

比較回数と交換回数の比率

比較による整列アルゴリズムでは（比較によらないソートアルゴリズムも存在し，ビンソートなどがある），その中心となる処理は

  • 2つのレコードのキーの大小を判定する（比較）
  • もし必要ならそれらを入れ換える（交換）
という2つの部分から成り立っています．

同じ計算量のアルゴリズムでも，比較回数と交換回数の比率が異なります． もし，レコードが大きければ（交換の対象となるデータが文字列や構造体など，そのサイズが大規模である場合）， 実際の計算時間は交換の回数に左右されることになります． その場合は，計算量が同じであれば，交換の回数が少ないアルゴリズムを選ぶのが望ましいです． 逆に，もしキーの比較にコストがかかるのであれば，計算量が同じであれば，できるだけ比較の回数が少ないアルゴリズムを選ぶべきです．

なお，交換の対象となるレコード（データ）そのものを配列に格納すると， そのレコードが文字列や構造体など，そのサイズが大規模である場合， 単純な交換は非効率となります． そのため，上記で述べた通り，交換の回数に実際の計算時間が左右されやすくなります． そこで，配列にソート対象となるレコードそのものを格納するのではなく， 対象となるレコードへのポインタを記憶するようにします． すると，交換の際には，データそのものは一切移動せずに， そのデータへのポインタを交換することになり，効率的なソートの実行が可能となります．

定数項部分

計算量だけでなく，定数項部分にも注意を払う必要があります．高速なアルゴリズムは技巧をこらして処理が複雑になる傾向があり， 単純なアルゴリズムに比べて計算量の定数項部分が大きくなります．ですから，対象となるレコード数があまり多くなければ， 高速なアルゴリズムよりも単純なアルゴリズムのほうが処理が速いことがあります．

一般に，nnがごく小さい場合（数十程度）には，単純なアルゴリズムを使い，nnが1000個程度までならシェルソートを使い，それ以上ならばクイックソートを使うのが良いとされています． また，レコードが多すぎる場合には，マージソートなどを利用することになります．

ソートアルゴリズムの分類

ソートアルゴリズムはいくつかの視点から分類が可能です．

安定なソート

同じキーのソート前の順序が，ソート後も同じになるようなソートアルゴリズムを安定ソート（stable sort）といいます． 例えば，学籍番号順に並んでいる試験結果を試験点数の降順でソートしたときに，同じ得点の学生は学籍番号順に並ぶことになります． 安定ソートの例としては，バブルソート，挿入ソート，マージソートなどがあります．

一方，安定でないアルゴリズムでは，ソート前の位置関係は保存されません．なお，常に位置関係が保たれないわけでもないということにも注意が必要です． つまり，安定でないアルゴリズムでは，ソート前の位置関係が保たれるように並べる場合もあれば，そうでない場合もあり， それらのどちらになるかはあらかじめ予測できません．安定でないアルゴリズムの例としては，シェルソートやクイックソートなどがあります．

一般に，単純な（遅い）アルゴリズムは安定で，複雑な（高速な）アルゴリズムは安定でないという傾向があります（が，例外もあります）．

内部ソートと外部ソート

ソートされるデータの格納領域（メモリ）上で並び替えを進めるソートアルゴリズムを内部ソート（internal sorting）といいます． 対して，ソートされるデータの格納領域以外に同等以上の記憶領域を必要とするソートアルゴリズムを外部ソート（external sorting）といいます． 外部ソートの代表例として，マージソートがあります． メモリに乗らないような巨大なデータをソートするにはマージソートのような外部ソートが利用されます．


Collectionsクラス

Collectionsクラス（復習：第19章 コレクション）には，List, Set, Mapなどのデータ構造を操作するためのメソッドが実装されています． これらのメソッドはクラスメソッド（復習：第11章 クラスとインスタンス）として実装されているので，Collectionsクラスのインスタンスを作ることなく，利用することができます．

Collections.sortメソッド

Collectionsのクラスメソッドの中にはリストをソートするsortメソッドがあります． Collectionsクラスのsortメソッドはマージソートが実装されており，計算量O(nlogn)O(nlog⁡n)のソートを利用することができます（参照）． sortメソッドでは，指定されたリストを要素の「自然順序付け」に従って，昇順にソートします．

ここで，ソートをするためには，順序付けをどうするかが明確に決まっている必要があります． 整数を扱う場合は，その順序は数値の大小で決まります． Collections.sortメソッドでは，リストになっている任意のオブジェクトをソートすることができますが，任意の2つのオブジェクトを比較したときに，その前後が明確に決まっていなくてはなりません． この順序付けの方法には2通りの方法があります．

  • ソート対象のデータ自身が，自分と他のオブジェクトとの比較をして，順序を決められるようにする
  • ソートする順序を決めた特別なクラスを用意する

前者をComparableインタフェース，後者をComparatorインタフェースといいます．（なお，インタフェースについては、「第14章 インタフェース」を復習ください．）

Comparableインタフェース

1つ目の方法は，クラスにComparableインタフェースを実装する方法です． これは自分自身と他のオブジェクトを比較するcompareToメソッドを実装することで，オブジェクト間の順番を明確にするというものです． 例えば，StringクラスはComparable<String>インタフェースを実装しているので，compareToメソッドがあります．

compareToメソッドは戻り値としてintを返します． 自分自身と他のオブジェクトを比較した結果，等しければ0，自分が相手よりも順序が前なら負（-1），順序が後なら正（1）が戻り値となります． Collections.sortを使う例を以下に示します． 

package ex2;

import java.util.ArrayList;
import java.util.Collections;

public class SampleSort {
  public static void main(String[] args) {
    ArrayList<String> list = new ArrayList<String>();
    
    list.add("dave");
    list.add("bob");
    list.add("alice");
    list.add("carol");
    list.add("10");
    list.add("2");
    list.add("100");
    list.add("11");
    
    System.out.println("ソート前");
    for(String str: list) {
      System.out.println(str);
    }
    
    Collections.sort(list);
    
    System.out.println("ソート後");
    for(String str: list) {
      System.out.println(str);
    }
  }
}

自分で定義したクラスであっても，同様にComparableインタフェースでcompareToメソッドを実装すれば，Collections.sortメソッドによるソートを利用することができます． 

package ex2;

import java.util.ArrayList;
import java.util.Collections;

public class SortedMusicListenerWithComparable {
  public static void main(String[] args) {
    ArrayList<Music> album = new ArrayList<Music>();

    album.add(new Music("Technopolis", "Yellow Magic Orchestra"));
    album.add(new Music("Absolute Ego Dance", "Yellow Magic Orchestra"));
    album.add(new Music("Rydeen", "Yellow Magic Orchestra"));
    album.add(new Music("Castalia", "Yellow Magic Orchestra"));
    album.add(new Music("Behind the Mask", "Yellow Magic Orchestra"));
    album.add(new Music("Day Tripper", "Yellow Magic Orchestra"));
    album.add(new Music("Insomnia", "Yellow Magic Orchestra"));
    album.add(new Music("Solid State Survivor", "Yellow Magic Orchestra"));

    Collections.sort(album);

    for (Music music : album) {
      System.out.println(music.getTitle() + " by " + music.getArtist());
    }
  }
}

class Music implements Comparable<Music> {
  private String title;
  private String artist;

  public Music(String title, String artist) {
    this.title = title;
    this.artist = artist;
  }
  public String getTitle() {
    return title;
  }
  public String getArtist() {
    return artist;
  }
  // タイトルで比較
  public int compareTo(Music another) {
    String anotherTitle = another.getTitle();
    if (title.compareTo(anotherTitle) < 0) {
      // 自分が相手よりも前
      return -1;
    } else if (title.compareTo(anotherTitle) > 0) {
      // 自分が相手よりも後
      return 1;
    } else {
      // それ以外＝同一順位
      return 0;
    }
  }
}

Comparatorインタフェース

もう1つの方法は，順序付けを決める専用のクラスを準備する方法です． このクラスはComparatorインタフェースを実装し，2つのオブジェクトを引数とするcompareメソッドを実装します． 戻り値はcompareToメソッドと同様です． この方法はデータを表すクラスが変更できなくても，別のクラスを作成すれば良いので，ソートの順序付けを定義することができます． また，順序付けの方法が後から変更になる可能性がある場合も，別クラスなので有効に利用できます． 

package ex2;

import java.util.ArrayList;
import java.util.Collections;
import java.util.Comparator;

public class SortedMusicListenerWithComparator {
  public static void main(String[] args) {
    ArrayList<Music2> album = new ArrayList<Music2>();

    album.add(new Music2("Technopolis", "Yellow Magic Orchestra"));
    album.add(new Music2("Absolute Ego Dance", "Yellow Magic Orchestra"));
    album.add(new Music2("Rydeen", "Yellow Magic Orchestra"));
    album.add(new Music2("Castalia", "Yellow Magic Orchestra"));
    album.add(new Music2("Behind the Mask", "Yellow Magic Orchestra"));
    album.add(new Music2("Day Tripper", "Yellow Magic Orchestra"));
    album.add(new Music2("Insomnia", "Yellow Magic Orchestra"));
    album.add(new Music2("Solid State Survivor", "Yellow Magic Orchestra"));

    Collections.sort(album, new MusicComparator());

    for (Music2 music : album) {
      System.out.println(music.getTitle() + " by " + music.getArtist());
    }
  }
}

class Music2 {
  private String title;
  private String artist;

  public Music2(String title, String artist) {
    this.title = title;
    this.artist = artist;
  }
  public String getTitle() {
    return title;
  }
  public String getArtist() {
    return artist;
  }
}
class MusicComparator implements Comparator<Music2> {
  public int compare(Music2 music1, Music2 music2) {
    return music1.getTitle().compareTo(music2.getTitle());
  }
}
